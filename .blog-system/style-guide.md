# 블로그 스타일 가이드

이 가이드는 2024년 이후 회고/에세이 글 10개를 심층 분석하여 추출한 글쓰기 스타일입니다.

---

## 1. 문체 (Tone & Voice)

### 기본 톤

- **솔직하고 성찰적인 구어체**: 마치 친한 친구에게 속마음을 털어놓듯 진솔하게 서술
- **1인칭 내러티브 유지**: "나는", "내가", "우리"를 자연스럽게 사용
- **감정의 깊이를 담은 서술**: 단순 사실 나열이 아닌 그 순간의 감정과 생각을 함께 기록
- **자기 비판적이면서도 희망적**: 실패와 부족함을 인정하면서도 다시 일어서겠다는 의지 표현

### 어조의 특징

| 상황 | 어조 예시 |
|------|----------|
| 어려움 서술 | "결국 한계가 왔다", "마음이 힘들어진다" |
| 성찰 | "지나고 보니", "돌아보면", "생각해보면" |
| 다짐 | "~하기로 했다", "~해보자", "~하려고 한다" |
| 솔직한 고백 | "솔직히", "사실", "참 못난 나 자신이다" |
| 유머/자조 | "하하, 맞다", "주절주절 칭얼칭얼", "그런 건 안비밀이다" |

### 자주 사용하는 표현

#### 전환어
- **"아무튼"** - 화제 정리, 샛길에서 복귀
- **"그래서"** - 인과관계 연결
- **"다만"** - 예외 조건, 반론 제시
- **"그런데"** - 맥락 전환, 반전
- **"결국"** - 결론 도출, 귀결점 제시
- **"솔직히"** - 진솔한 의견 시작
- **"여하튼"** / **"여튼"** - 화제 정리
- **"각설하고"** - 본론으로 강하게 전환 (샛길 정리 후)
- **"여튼 각설하고"** - 복합 전환어, 강조된 본론 복귀
- **"생각해보면"** - 성찰 시작
- **"다시 ~로 돌아가 보자"** - 이전 맥락 복귀
- **"그렇다면"** - 논리적 연결, 질문 제기

#### 종결어 패턴
- **"~라고 본다"** / **"~라고 생각한다"** - 주관적 판단
- **"~했다"**, **"~이었다"** - 과거 서술 (기본형)
- **"~인 것 같다"** - 완화된 주장
- **"~뿐이다"** - 강조, 한정
- **"~하기로 했다"** - 결심 표현
- **"~나 보다"** - 깨달음, 자각
- **"~할 것이다"** - 미래 다짐

#### 특징적 표현 패턴
- 괄호 안 부연/감정 삽입: "(지나고보니 10년이다.)", "(언제가 될지 모르겠지만)"
- 자문자답: "음, 그런가?", "뭐가 최선일까?", "이런 경험 없나?"
- 반어/자조: "그냥 열심히 하려고 하다가 그런 건데"
- 영화/책 인용 삽입: 대화체로 인용 후 출처 명시
- 어그로성 도입: 자극적 제목 후 괄호로 실제 상황 해명
- 비유/메타포 활용: 친숙한 개념으로 기술 설명 (페어 프로그래밍, 오케스트라, 레고, 운전, 스포츠팀/축구)

### 감정 표현 스펙트럼

```
좌절/힘듦:
- "마음이 힘들어진다"
- "무력하고, 무기력하고"
- "결국 한계가 왔다"
- "참 슬프다"
- "~라는 건 참 서글픈 일이다"
- "뼈저리게 공감했다"
- "지금 생각하면 얼굴이 화끈거린다"

성찰/깨달음:
- "지나고 보니"
- "돌아보면"
- "이제야 깨달았다"
- "아쉬움이 남는다"
- "그때 깨달았다"
- "그때 배웠다"

희망/다짐:
- "그래도 아직 불씨가 꺼지지 않았다"
- "다시 일어서면 된다"
- "새로운 출발을 해보자"
- "미래는 바뀔 수 있다"
- "다시 시작하는 마음으로"
```

### 피해야 할 표현

- 과도한 경어체 ("~하십시오", "~입니다" 반복)
- 기계적인 나열 ("첫째, 둘째, 셋째")
- 비즈니스 클리셰 ("시너지", "레버리지", "패러다임 시프트")
- 과도한 이모지/이모티콘
- 딱딱한 학술체나 보고서체
- 감정 없는 건조한 서술

## 2. 구조 (Structure)

### 회고 글 기본 구조

```markdown
## 들어가며
- 회고 배경 설명 (왜 이 시점에 쓰는지)
- 전체적인 톤 설정
- 1-2 문단

## 본문 섹션들 (주제별 또는 시간순)
### 섹션 제목
- 사실(what happened) + 감정(how I felt) + 성찰(what I learned)
- 구체적인 에피소드 중심

## 나가며
- 전체 회고 요약
- 앞으로의 다짐
- 인용문으로 여운 남기기 (선택)
```

### 소제목 스타일

- `## 대제목` - 주요 섹션 구분 (들어가며, 나가며, 핵심 주제)
- `### 소제목` - 세부 주제, 시기별 구분
- **명사형 선호**: "결혼 생활", "회사", "개인"
- **질문형도 효과적**: "그래서 뭘 배웠나?"

### 단락 구성 특징

1. **긴 문단 + 짧은 문장 마무리**
   - 상황 설명하는 긴 문단 후
   - "그게 다였다.", "참 어려웠다." 같은 짧은 문장으로 리듬

2. **감정 표현 단독 문단**
   - 중요한 감정이나 깨달음은 별도 단락으로 분리
   - "마음이 힘들어진다." 단독으로 배치

3. **리스트 활용**
   - 월별 요약, 액션 아이템에 리스트 사용
   - 본문 서술은 산문 중심

## 3. 콘텐츠 패턴

### 회고 글 (월간/기간/프로젝트)

**핵심 요소:**
- 시간적 맥락 제시 (언제부터 언제까지)
- 사건 + 감정 + 성찰의 삼박자
- 솔직한 실패/어려움 인정
- 인용문으로 마무리

**구조 예시:**
```markdown
## 들어가며
(왜 이 회고를 쓰는지, 배경)

## 회사 / 프로젝트 섹션
### 월별 또는 주제별 소제목
(사건 서술 + 그때의 감정 + 배운 점)

## 개인 섹션
(일 외적인 삶: 건강, 독서, 관계 등)

## 나가며
(다짐, 액션 아이템, 인용문)
```

### 에세이 글

**핵심 요소:**
- 인용문/에피소드로 시작
- 개인 경험과 연결
- 보편적 통찰로 확장
- 여운 있는 마무리

**구조 예시:**
```markdown
## 인용문 또는 에피소드 (도입)
> 인용문
- 출처

---

## 들어가며
(왜 이 주제에 대해 쓰는지)

## 본론 (경험과 성찰)
### 소제목
(개인 경험 + 깨달음)

## 나가며
(결론, 다짐, 마무리 인용문)
```

### 분석 에세이 글 (외부 사례 + 개인 경험)

**핵심 요소:**
- 외부 인물/사례를 분석하면서 개인 경험과 연결
- 인용문 제시 후 바로 개인 성찰로 연결
- 각 섹션 끝에 액션 아이템 정리
- 마무리에 테이블로 전체 요약
- 영어 인용문으로 여운 남기기

**구조 예시:**
```markdown
## 들어가며
(왜 이 주제/인물에 관심을 갖게 되었는지)
(개인적 배경과 연결점)

## 본론 섹션들
### 주제별 소제목
> 외부 인용문
> — 출처

(인용문에 대한 분석)

(개인 경험과 연결: "나도 ~했었는데", "내가 ~하면서 느낀 건")

**액션 아이템:**
1. 구체적인 실천 항목
2. 구체적인 실천 항목

---

## 나가며
(전체 요약)

| 배운 것 | 액션 아이템 |
|---------|------------|
| 핵심 교훈 1 | 실천 항목 1 |
| 핵심 교훈 2 | 실천 항목 2 |

> 영어 인용문
> 한글 번역
>
> — 출처

(마무리 한 문장)
```

**특징적 패턴:**
- 외부 인용 -> 분석 -> 개인 경험 -> 액션 아이템의 흐름
- 괄호 안 자조적 코멘트: "(아니, 엄청 어렵다.)", "(팬심 전환 속도가 이렇게 빠를 줄이야.)"
- 솔직한 자기 성찰: "솔직히 ~는 정말 어렵다", "나도 ~한 적이 있었다"
- 질문형 성찰: "~하지 않았을까 싶다", "~했으면 더 좋았을텐데"

### 논증 에세이 글 (심층 탐구)

**핵심 요소:**
- 외부 인용문/인터뷰로 강렬하게 시작
- 개인 경험과 깊이 연결
- 여러 관점에서 문제 탐구 (리더 시점, 팀원 시점 등)
- 스포츠/비즈니스 비유 적극 활용
- 구체적 실천 방안 제시
- 에필로그/후기로 개인적 마무리

**구조 예시:**
```markdown
## 들어가며
> 외부 인용문 (인터뷰, 책 등)

(인용문에 대한 감정적 반응)
(개인 경험과 연결)

---

## 문제 제기 섹션
(왜 이 문제가 중요한지)

## 관점 1: 소제목
(한 측면에서의 분석 + 개인 경험)

## 관점 2: 소제목
(다른 측면에서의 분석 + 개인 경험)

## 관점 N: 소제목
...

## 나가며
(결론, 질문으로 마무리)

---

## 실천 방안 섹션
### 1. 구체적 실천 항목
(설명)

### 2. 구체적 실천 항목
(설명)

---

### 에필로그/개인적 후기
(회고 대신 이 글로 대체한다는 언급)
(개인적 소회, 기도문 형식 등)

> 영어 인용문
> 한글 번역
>
> — 출처
```

**특징적 패턴:**
- 인용문 후 "뼈저리게 공감했다", "한동안 멍하니 ~할 수밖에 없었다" 등 강한 감정 반응
- 스포츠 비유: 감독-선수, 골대, 필드, 드리블, 공격수-수비수
- 질문으로 마무리: "나는 지금 승리하고 있는가?"
- 에필로그에서 인용 블록(>)으로 개인적 후기 작성

### 기술/프로젝트 회고 글

**핵심 요소:**
- 프로젝트 배경과 목표 설명
- 기술적 내용 + 개인적 경험 병행
- Keep/Problem/Try 프레임워크 활용
- Lesson Learned 정리

**구조 예시:**
```markdown
## 들어가며
(프로젝트 배경, 목표)

## 프로젝트 셋업
### 환경 / 기술 스택
(사용한 도구, 환경 설명)

## 회고
### Keep
(잘한 점)

### Problem
(문제점, 어려웠던 점)

### Try
(개선할 점, 다음에 시도할 것)

### Lesson Learned
- 불릿 포인트로 핵심 교훈 정리

## 나가며
(전체 요약, 다짐)
```

### 기술 에세이 글 (경험 기반)

**핵심 요소:**
- 어그로성 도입으로 관심 유도 (선택)
- 문제/회의감 제시 -> 영감의 원천 -> 해결책 -> 권유/마무리
- 기술 설명과 개인 감정/고민의 자연스러운 병행
- 친숙한 비유로 기술 개념 설명
- Mermaid 다이어그램, 코드 블록, 테이블 적극 활용

**구조 예시:**
```markdown
## 들어가며
> 어그로성 문장 (선택)
(배경 설명, 문제/회의감 제시)

## 영감의 원천
(참고한 글/책/인물 소개, 핵심 개념 설명)

## 문제 분석
(왜 기존 방식이 불편했는지, 개인적 고민)

## 해결책/나만의 방법
### 접근 방식
(어떻게 해결했는지, 워크플로우 설명)

### 실제 구조/예시
(코드 블록, Mermaid 다이어그램, 테이블 활용)

## 시작하는 방법 (권유)
(독자가 따라할 수 있는 가이드)

## 나가며
(핵심 메시지, 본질적 가치 강조, 다짐)
```

**시각 자료 활용 가이드:**

| 유형 | 용도 | 예시 |
|------|------|------|
| Mermaid flowchart | 워크플로우, 프로세스 | TDD 사이클, 리뷰 흐름 |
| 코드 블록 (tree) | 폴더/파일 구조 | 프로젝트 구조 설명 |
| 코드 블록 (markdown) | 설정 파일, 커맨드 | plan.md 예시 |
| ASCII 다이어그램 | 간단한 구조 | 병렬 처리 흐름 |
| 테이블 | 비교, 분류, Tier | 리뷰 강도별 정리 |

## 4. 특징적 패턴

### 어그로 도입 패턴

독자의 관심을 끄는 자극적 시작 후 괄호로 실제 상황을 해명하는 자조적 유머:

```
> 먼저 어그로 끌어서 미안하다. 나는 15년차 CTO가 아니다.
> (15년째 개발을 하고 있는 건 맞음)
> 그 다음 현재 CTO도 아니다. (작년에 퇴사함.)
```

**활용 시점:**
- 기술 에세이나 경험 공유 글의 도입부
- 제목이 자극적인 경우 해명
- 독자와의 거리감을 좁히고 싶을 때

### 비유/메타포 활용 패턴

복잡한 기술 개념을 친숙한 상황에 비유:

```
"일종의 페어 프로그래밍이랄까. AI가 키보드를 잡고, 나는 옆에서
'어, 거기 그렇게 하면 안 돼' 하면서 방향을 잡아주는 느낌이다."

"마치 오케스트라 지휘자처럼, 각 악기(에이전트)가 언제 어떤 역할을

**자주 사용하는 비유 영역:**
- 협업/역할 분담: 페어 프로그래밍, 오케스트라

```

### 액션 아이템 형식

```
**액션 아이템:**
1. **"내가 할 수 있나?"가 아니라 "누가 제일 잘할 수 있나?"로 질문 바꾸기.**
2. **결정을 미루지 않기.** 정보가 70% 모이면 결정하고 나머지는 실행하면서 조정하기.
3. **위임 후 간섭하지 않기.** 결과를 리뷰하되, 과정에 일일이 개입하지 않기.
```
