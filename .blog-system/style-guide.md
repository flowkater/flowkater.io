# 블로그 스타일 가이드

이 가이드는 2024년 이후 회고/에세이 글 8개를 심층 분석하여 추출한 글쓰기 스타일입니다.

---

## 1. 문체 (Tone & Voice)

### 기본 톤

- **솔직하고 성찰적인 구어체**: 마치 친한 친구에게 속마음을 털어놓듯 진솔하게 서술
- **1인칭 내러티브 유지**: "나는", "내가", "우리"를 자연스럽게 사용
- **감정의 깊이를 담은 서술**: 단순 사실 나열이 아닌 그 순간의 감정과 생각을 함께 기록
- **자기 비판적이면서도 희망적**: 실패와 부족함을 인정하면서도 다시 일어서겠다는 의지 표현

### 어조의 특징

| 상황 | 어조 예시 |
|------|----------|
| 어려움 서술 | "결국 한계가 왔다", "마음이 힘들어진다" |
| 성찰 | "지나고 보니", "돌아보면", "생각해보면" |
| 다짐 | "~하기로 했다", "~해보자", "~하려고 한다" |
| 솔직한 고백 | "솔직히", "사실", "참 못난 나 자신이다" |
| 유머/자조 | "하하, 맞다", "주절주절 칭얼칭얼", "그런 건 안비밀이다" |

### 자주 사용하는 표현

#### 전환어
- **"아무튼"** - 화제 정리, 샛길에서 복귀
- **"그래서"** - 인과관계 연결
- **"다만"** - 예외 조건, 반론 제시
- **"그런데"** - 맥락 전환, 반전
- **"결국"** - 결론 도출, 귀결점 제시
- **"솔직히"** - 진솔한 의견 시작
- **"여하튼"** / **"여튼"** - 화제 정리
- **"각설하고"** - 본론으로 전환

#### 종결어 패턴
- **"~라고 본다"** / **"~라고 생각한다"** - 주관적 판단
- **"~했다"**, **"~이었다"** - 과거 서술 (기본형)
- **"~인 것 같다"** - 완화된 주장
- **"~뿐이다"** - 강조, 한정
- **"~하기로 했다"** - 결심 표현
- **"~나 보다"** - 깨달음, 자각
- **"~할 것이다"** - 미래 다짐

#### 특징적 표현 패턴
- 괄호 안 부연/감정 삽입: "(지나고보니 10년이다.)", "(언제가 될지 모르겠지만)"
- 자문자답: "음, 그런가?", "뭐가 최선일까?"
- 반어/자조: "그냥 열심히 하려고 하다가 그런 건데"
- 영화/책 인용 삽입: 대화체로 인용 후 출처 명시

### 감정 표현 스펙트럼

```
좌절/힘듦:
- "마음이 힘들어진다"
- "무력하고, 무기력하고"
- "결국 한계가 왔다"
- "참 슬프다"

성찰/깨달음:
- "지나고 보니"
- "돌아보면"
- "이제야 깨달았다"
- "아쉬움이 남는다"

희망/다짐:
- "그래도 아직 불씨가 꺼지지 않았다"
- "다시 일어서면 된다"
- "새로운 출발을 해보자"
```

### 피해야 할 표현

- 과도한 경어체 ("~하십시오", "~입니다" 반복)
- 기계적인 나열 ("첫째, 둘째, 셋째")
- 비즈니스 클리셰 ("시너지", "레버리지", "패러다임 시프트")
- 과도한 이모지/이모티콘
- 딱딱한 학술체나 보고서체
- 감정 없는 건조한 서술

## 2. 구조 (Structure)

### 회고 글 기본 구조

```markdown
## 들어가며
- 회고 배경 설명 (왜 이 시점에 쓰는지)
- 전체적인 톤 설정
- 1-2 문단

## 본문 섹션들 (주제별 또는 시간순)
### 섹션 제목
- 사실(what happened) + 감정(how I felt) + 성찰(what I learned)
- 구체적인 에피소드 중심

## 나가며
- 전체 회고 요약
- 앞으로의 다짐
- 인용문으로 여운 남기기 (선택)
```

### 소제목 스타일

- `## 대제목` - 주요 섹션 구분 (들어가며, 나가며, 핵심 주제)
- `### 소제목` - 세부 주제, 시기별 구분
- **명사형 선호**: "결혼 생활", "회사", "개인"
- **질문형도 효과적**: "그래서 뭘 배웠나?"

### 단락 구성 특징

1. **긴 문단 + 짧은 문장 마무리**
   - 상황 설명하는 긴 문단 후
   - "그게 다였다.", "참 어려웠다." 같은 짧은 문장으로 리듬

2. **감정 표현 단독 문단**
   - 중요한 감정이나 깨달음은 별도 단락으로 분리
   - "마음이 힘들어진다." 단독으로 배치

3. **리스트 활용**
   - 월별 요약, 액션 아이템에 리스트 사용
   - 본문 서술은 산문 중심

## 3. 콘텐츠 패턴

### 회고 글 (월간/기간/프로젝트)

**핵심 요소:**
- 시간적 맥락 제시 (언제부터 언제까지)
- 사건 + 감정 + 성찰의 삼박자
- 솔직한 실패/어려움 인정
- 인용문으로 마무리

**구조 예시:**
```markdown
## 들어가며
(왜 이 회고를 쓰는지, 배경)

## 회사 / 프로젝트 섹션
### 월별 또는 주제별 소제목
(사건 서술 + 그때의 감정 + 배운 점)

## 개인 섹션
(일 외적인 삶: 건강, 독서, 관계 등)

## 나가며
(다짐, 액션 아이템, 인용문)
```

### 에세이 글

**핵심 요소:**
- 인용문/에피소드로 시작
- 개인 경험과 연결
- 보편적 통찰로 확장
- 여운 있는 마무리

**구조 예시:**
```markdown
## 인용문 또는 에피소드 (도입)
> 인용문
- 출처

---

## 들어가며
(왜 이 주제에 대해 쓰는지)

## 본론 (경험과 성찰)
### 소제목
(개인 경험 + 깨달음)

## 나가며
(결론, 다짐, 마무리 인용문)
```

### 기술/프로젝트 회고 글

**핵심 요소:**
- 프로젝트 배경과 목표 설명
- 기술적 내용 + 개인적 경험 병행
- Keep/Problem/Try 프레임워크 활용
- Lesson Learned 정리

**구조 예시:**
```markdown
## 들어가며
(프로젝트 배경, 목표)

## 프로젝트 셋업
### 환경 / 기술 스택
(사용한 도구, 환경 설명)

## 회고
### Keep
(잘한 점)

### Problem
(문제점, 어려웠던 점)

### Try
(개선할 점, 다음에 시도할 것)

### Lesson Learned
- 불릿 포인트로 핵심 교훈 정리

## 나가며
(전체 요약, 다짐)
```

## 4. 특징적 패턴

### 문장 리듬 패턴

**긴 문장 후 짧은 문장:**
```
"9월은 회사에서 B2G 사업이 일단락되는 시즌이었다. 지나고 보니 여름이 더운지도
모르고 일했던 그 시절이었는데, 그저 당시에는 끝났다는 안도감에 잠깐의 휴식을
가지고 싶었다."

"그게 다였다."
```

**반복 구문 활용:**
```
"무력하고, 무기력하고 할 수 있는 최선을 다하고자 마음을 다잡아도 결국 무너지고 만다."
```

### 인용문 활용 패턴

**영어-한글 병기:**
```markdown
> All that is gold does not glitter
> 금이라고 다 반짝이진 않으며
> Not all those who wander are lost
> 방랑자라고 다 길 잃은 것은 아니다

- J.R.R. 톨킨, 반지의 제왕 중
```

**드라마/영화 대사 인용:**
```markdown
> "일은 좋아해요?"
> "글쎄요, 사람들은 좋아해요."

*더 베어 시즌3 중*
```

### 강조 방식

- **굵게** - 핵심 키워드, 중요 개념
- 단독 단락 - 감정적으로 중요한 문장
- 인용 블록 (>) - 타인 말, 책/영화 구절
- 이탤릭 (_텍스트_) - 이미지 캡션, 부가 설명

### 마무리 패턴

**인용문 + 다짐형:**
```markdown
> 인용문
> 한글 번역

- 출처

---

앞으로의 구체적인 계획이나 다음 글 예고
```

**자기 위로형:**
```markdown
주절주절 칭얼칭얼 말이 너무 길었다.
감상에 그만 젖고 이제 다시 Let's hit it.
```

**액션 아이템형:**
```markdown
**Try**

* 침착하게 해야할 것을 하나씩 해나가기
* 조직 동기부여, 업무 파이프라인 개선 포기하지 않기
* 꺾여도 계속가는 마음
```

### 글 연결 패턴

**이전/다음 글 링크:**
```markdown
---

* 이전 회고
  * [이전 글 제목](/posts/이전-글-slug)
* 다음 회고
  * [다음 글 제목](/posts/다음-글-slug)
```

## 5. Frontmatter 템플릿

```yaml
---
title: "주제목 - 부제목 또는 (기간)"
description: "SEO용 1-2문장 요약. 핵심 키워드 포함."
summary: "글 내용 3-4문장 미리보기. 독자가 글을 읽을지 판단할 수 있는 정보."
pubDatetime: YYYY-MM-DDTHH:mm:ss+09:00
updatedDatetime: YYYY-MM-DDTHH:mm:ss+09:00
tags: ["회고", "월간회고"]  # 또는 ["에세이", "리더십"]
featured: false  # 대표 글인 경우 true
draft: false
---
```

### 태그 체계

| 대분류 | 세부 태그 | 용도 |
|--------|----------|------|
| 회고 | 월간회고, 기간회고, 프로젝트회고 | 시간/프로젝트 단위 돌아보기 |
| 에세이 | 리더십, 커리어 | 생각과 통찰 |
| 기술 | 특정 기술명 (scrumble, golang 등) | 개발 관련 |
| 독서 | - | 책 리뷰 |

### 제목 패턴

- **회고**: "YYYY년 M월 회고", "프로젝트명 회고 (기간)"
- **에세이**: "주제 - 영문 부제" 또는 인상적인 문구
- **기술**: "프로젝트 기술 회고 - 세부 주제"

## 7. 품질 체크리스트

작성 후 확인:

### 문체
- [ ] 1인칭 시점이 일관되게 유지되는가?
- [ ] 사실과 감정이 균형 있게 서술되었는가?
- [ ] 과도하게 건조하거나 감상적이지 않은가?
- [ ] 전환어가 자연스럽게 배치되었는가?

### 구조
- [ ] 들어가며-본문-나가며 구조가 갖춰졌는가?
- [ ] 소제목으로 적절히 구분되었는가?
- [ ] 마무리에 여운/다짐이 있는가?

### 특징
- [ ] 인용문이 적절히 활용되었는가? (선택)
- [ ] 이전/다음 글 링크가 있는가? (회고 시리즈)
- [ ] 이모지를 과다 사용하지 않았는가?

### Frontmatter
- [ ] title, description, summary가 작성되었는가?
- [ ] 날짜가 올바른 형식인가? (YYYY-MM-DDTHH:mm:ss+09:00)
- [ ] 적절한 태그가 지정되었는가?
- [ ] draft가 false로 설정되었는가? (게시 시)

## 8. 예시 문장 모음

### 도입부

```
"8월 이후로 회고를 전혀 하지 않았다. 몸도 마음도 건강하지 않은 상태에서
9월부터 3월까지 반년 이상이 지나갔다."

"나이를 먹고 경험이 쌓이고 경력이 쌓이면서 생기는 습관 중 하나가
제한적인 상황에서 빠르게 판단하는 습관이다."

"처음으로 법원을 방문했다."
```

### 감정 표현

```
"당시에는 이런 내 입장을 모르는 팀원들도, 경영진도 전부 싫었다."

"알량한 책임감 하나로 여기까지 왔는데"

"참 못난 나 자신이다."

"그래도 그 불씨가 아예 꺼지지는 않았다고 믿고 있다."
```

### 성찰/깨달음

```
"지나고 보니 그렇게 많은 일을 못한 것 같기도 하고 또 지난 시간을
곰곰히 곱씹어보면 무엇하나 쉬웠던 시기는 없었다."

"결국 내가 할 수 있는 최선은 이것뿐이다."

"인생은 항상 쉬운 결정과 어려운 결정을 쥐어주고 하나를 선택하라고 강요한다.
그래서 가혹하다. 그런데 항상 정답은 어려운 결정이었다."
```

### 마무리

```
"4년간의 치열했던 기억은 여기 묻어두고 좋은 기억과 배움만 손에 쥐고
새로운 출발을 해보자."

"언젠가 또 생각이 많은 날이 온다면, 다시 이 글을 돌이켜 보며 되새기자."

"감상에 그만 젖고 이제 다시 Let's hit it."
```
