# 블로그 스타일 가이드

이 가이드는 2024년 이후 회고/에세이 글 10개를 심층 분석하여 추출한 글쓰기 스타일입니다.

---

## 1. 문체 (Tone & Voice)

### 기본 톤

- **솔직하고 성찰적인 구어체**: 마치 친한 친구에게 속마음을 털어놓듯 진솔하게 서술
- **1인칭 내러티브 유지**: "나는", "내가", "우리"를 자연스럽게 사용
- **감정의 깊이를 담은 서술**: 단순 사실 나열이 아닌 그 순간의 감정과 생각을 함께 기록
- **자기 비판적이면서도 희망적**: 실패와 부족함을 인정하면서도 다시 일어서겠다는 의지 표현

### 어조의 특징

| 상황 | 어조 예시 |
|------|----------|
| 어려움 서술 | "결국 한계가 왔다", "마음이 힘들어진다" |
| 성찰 | "지나고 보니", "돌아보면", "생각해보면" |
| 다짐 | "~하기로 했다", "~해보자", "~하려고 한다" |
| 솔직한 고백 | "솔직히", "사실", "참 못난 나 자신이다" |
| 유머/자조 | "하하, 맞다", "주절주절 칭얼칭얼", "그런 건 안비밀이다" |

### 자주 사용하는 표현

#### 전환어
- **"아무튼"** - 화제 정리, 샛길에서 복귀
- **"그래서"** - 인과관계 연결
- **"다만"** - 예외 조건, 반론 제시
- **"그런데"** - 맥락 전환, 반전
- **"결국"** - 결론 도출, 귀결점 제시
- **"솔직히"** - 진솔한 의견 시작
- **"여하튼"** / **"여튼"** - 화제 정리
- **"각설하고"** - 본론으로 강하게 전환 (샛길 정리 후)
- **"여튼 각설하고"** - 복합 전환어, 강조된 본론 복귀
- **"생각해보면"** - 성찰 시작
- **"다시 ~로 돌아가 보자"** - 이전 맥락 복귀
- **"그렇다면"** - 논리적 연결, 질문 제기

#### 종결어 패턴
- **"~라고 본다"** / **"~라고 생각한다"** - 주관적 판단
- **"~했다"**, **"~이었다"** - 과거 서술 (기본형)
- **"~인 것 같다"** - 완화된 주장
- **"~뿐이다"** - 강조, 한정
- **"~하기로 했다"** - 결심 표현
- **"~나 보다"** - 깨달음, 자각
- **"~할 것이다"** - 미래 다짐

#### 특징적 표현 패턴
- 괄호 안 부연/감정 삽입: "(지나고보니 10년이다.)", "(언제가 될지 모르겠지만)"
- **괄호 일반화 패턴**: 전문 용어 뒤에 괄호로 일반적 설명을 병기하여 독자 범위를 넓힘
  - 예: "기술 아티클(또는 깊이 읽어야 하는 글)", "나만의 메모(노션, 옵시디언, 여백 등)"
  - 용도: 제목이나 본문에서 특정 용어가 독자층을 제한할 수 있을 때, 괄호로 범용화
- 자문자답: "음, 그런가?", "뭐가 최선일까?", "이런 경험 없나?", "근데 정말 ~가능한가? 된다."
- 반어/자조: "그냥 열심히 하려고 하다가 그런 건데"
- 영화/책 인용 삽입: 대화체로 인용 후 출처 명시
- 어그로성 도입: 자극적 제목 후 괄호로 실제 상황 해명
- 비유/메타포 활용: 친숙한 개념으로 기술 설명 (페어 프로그래밍, 오케스트라, 레고, 운전, 스포츠팀/축구)
- **개인 습관 삽입**: 방법론/가이드 글에서 "내 경우에는", "나는 ~하면서", "지나고 보니" 등으로 개인 실천 경험을 짧게 삽입하여 권위와 친밀감 확보

#### 인용문 연결 표현 (책 리뷰용)
- **와닿음 표현**: "이 문장이 특히 와닿았다", "이 문장에서 ~기보다는, 내 모습이 그대로 보였다"
- **저항-수용**: "선뜻 받아들이기 어려웠다. 그런데 곰곰이 생각해 보면"
- **성찰 연결**: "~를 읽으면서 지난 ~를 돌아봤다", "이 구절을 읽으면서 ~를 생각했다"
- **수사적 질문**: "얼마나 많은 ~를 미루고, 외면하고, 나중으로 떠넘겼는가"

#### 회복/성찰 표현
- **역설적 깨달음**: "그러나 다행히 내가 알고 있었던 건", "오히려 ~한 것 같다"
- **전환점 표현**: "다행히도 ~", "이제 조금 정리가 되었다"
- **열린 결론**: "~가 아닐까", "그게 ~이 아닐까"

### 감정 표현 스펙트럼

```
좌절/힘듦:
- "마음이 힘들어진다"
- "무력하고, 무기력하고"
- "결국 한계가 왔다"
- "참 슬프다"
- "~라는 건 참 서글픈 일이다"
- "뼈저리게 공감했다"
- "지금 생각하면 얼굴이 화끈거린다"

성찰/깨달음:
- "지나고 보니"
- "돌아보면"
- "이제야 깨달았다"
- "아쉬움이 남는다"
- "그때 깨달았다"
- "그때 배웠다"

희망/다짐:
- "그래도 아직 불씨가 꺼지지 않았다"
- "다시 일어서면 된다"
- "새로운 출발을 해보자"
- "미래는 바뀔 수 있다"
- "다시 시작하는 마음으로"
```

### 피해야 할 표현

- 과도한 경어체 ("~하십시오", "~입니다" 반복)
- 기계적인 나열 ("첫째, 둘째, 셋째")
- 비즈니스 클리셰 ("시너지", "레버리지", "패러다임 시프트")
- 과도한 이모지/이모티콘
- 딱딱한 학술체나 보고서체
- 감정 없는 건조한 서술
- **인용 후 직접적 감정 명명** ("뜨끔했다", "찔렸다", "뼈아팠다" 등)
  - 작위적으로 느껴짐
  - 대신: 상황이나 생각을 통해 간접적으로 감정 드러내기

## 2. 구조 (Structure)

### 회고 글 기본 구조

```markdown
## 들어가며
- 회고 배경 설명 (왜 이 시점에 쓰는지)
- 전체적인 톤 설정
- 1-2 문단

## 본문 섹션들 (주제별 또는 시간순)
### 섹션 제목
- 사실(what happened) + 감정(how I felt) + 성찰(what I learned)
- 구체적인 에피소드 중심

## 나가며
- 전체 회고 요약
- 앞으로의 다짐
- 인용문으로 여운 남기기 (선택)
```

### 소제목 스타일

- `## 대제목` - 주요 섹션 구분 (들어가며, 나가며, 핵심 주제)
- `### 소제목` - 세부 주제, 시기별 구분
- **명사형 선호**: "결혼 생활", "회사", "개인"
- **질문형도 효과적**: "그래서 뭘 배웠나?"

### 단락 구성 특징

1. **긴 문단 + 짧은 문장 마무리**
   - 상황 설명하는 긴 문단 후
   - "그게 다였다.", "참 어려웠다." 같은 짧은 문장으로 리듬

2. **감정 표현 단독 문단**
   - 중요한 감정이나 깨달음은 별도 단락으로 분리
   - "마음이 힘들어진다." 단독으로 배치

3. **리스트 활용**
   - 월별 요약, 액션 아이템에 리스트 사용
   - 본문 서술은 산문 중심

## 3. 콘텐츠 패턴

### 회고 글 (월간/기간/프로젝트)

**핵심 요소:**
- 시간적 맥락 제시 (언제부터 언제까지)
- 사건 + 감정 + 성찰의 삼박자
- 솔직한 실패/어려움 인정
- 인용문으로 마무리

**구조 예시:**
```markdown
## 들어가며
(왜 이 회고를 쓰는지, 배경)

## 회사 / 프로젝트 섹션
### 월별 또는 주제별 소제목
(사건 서술 + 그때의 감정 + 배운 점)

## 개인 섹션
(일 외적인 삶: 건강, 독서, 관계 등)

## 나가며
(다짐, 액션 아이템, 인용문)
```

### 에세이 글

**핵심 요소:**
- 인용문/에피소드로 시작
- 개인 경험과 연결
- 보편적 통찰로 확장
- 여운 있는 마무리

**구조 예시:**
```markdown
## 인용문 또는 에피소드 (도입)
> 인용문
- 출처

---

## 들어가며
(왜 이 주제에 대해 쓰는지)

## 본론 (경험과 성찰)
### 소제목
(개인 경험 + 깨달음)

## 나가며
(결론, 다짐, 마무리 인용문)
```

### 분석 에세이 글 (외부 사례 + 개인 경험)

**핵심 요소:**
- 외부 인물/사례를 분석하면서 개인 경험과 연결
- 인용문 제시 후 바로 개인 성찰로 연결
- 각 섹션 끝에 액션 아이템 정리
- 마무리에 테이블로 전체 요약
- 영어 인용문으로 여운 남기기

**구조 예시:**
```markdown
## 들어가며
(왜 이 주제/인물에 관심을 갖게 되었는지)
(개인적 배경과 연결점)

## 본론 섹션들
### 주제별 소제목
> 외부 인용문
> — 출처

(인용문에 대한 분석)

(개인 경험과 연결: "나도 ~했었는데", "내가 ~하면서 느낀 건")

**액션 아이템:**
1. 구체적인 실천 항목
2. 구체적인 실천 항목

---

## 나가며
(전체 요약)

| 배운 것 | 액션 아이템 |
|---------|------------|
| 핵심 교훈 1 | 실천 항목 1 |
| 핵심 교훈 2 | 실천 항목 2 |

> 영어 인용문
> 한글 번역
>
> — 출처

(마무리 한 문장)
```

**특징적 패턴:**
- 외부 인용 -> 분석 -> 개인 경험 -> 액션 아이템의 흐름
- 괄호 안 자조적 코멘트: "(아니, 엄청 어렵다.)", "(팬심 전환 속도가 이렇게 빠를 줄이야.)"
- 솔직한 자기 성찰: "솔직히 ~는 정말 어렵다", "나도 ~한 적이 있었다"
- 질문형 성찰: "~하지 않았을까 싶다", "~했으면 더 좋았을텐데"

### 논증 에세이 글 (심층 탐구)

**핵심 요소:**
- 외부 인용문/인터뷰로 강렬하게 시작
- 개인 경험과 깊이 연결
- 여러 관점에서 문제 탐구 (리더 시점, 팀원 시점 등)
- 스포츠/비즈니스 비유 적극 활용
- 구체적 실천 방안 제시
- 에필로그/후기로 개인적 마무리

**구조 예시:**
```markdown
## 들어가며
> 외부 인용문 (인터뷰, 책 등)

(인용문에 대한 감정적 반응)
(개인 경험과 연결)

---

## 문제 제기 섹션
(왜 이 문제가 중요한지)

## 관점 1: 소제목
(한 측면에서의 분석 + 개인 경험)

## 관점 2: 소제목
(다른 측면에서의 분석 + 개인 경험)

## 관점 N: 소제목
...

## 나가며
(결론, 질문으로 마무리)

---

## 실천 방안 섹션
### 1. 구체적 실천 항목
(설명)

### 2. 구체적 실천 항목
(설명)

---

### 에필로그/개인적 후기
(회고 대신 이 글로 대체한다는 언급)
(개인적 소회, 기도문 형식 등)

> 영어 인용문
> 한글 번역
>
> — 출처
```

**특징적 패턴:**
- 인용문 후 "뼈저리게 공감했다", "한동안 멍하니 ~할 수밖에 없었다" 등 강한 감정 반응
- 스포츠 비유: 감독-선수, 골대, 필드, 드리블, 공격수-수비수
- 질문으로 마무리: "나는 지금 승리하고 있는가?"
- 에필로그에서 인용 블록(>)으로 개인적 후기 작성

### 책 리뷰 에세이 글 (독서 + 성찰)

**핵심 요소:**
- 개인적 배경과 책을 읽게 된 계기 연결
- 책의 핵심 프레임워크/메시지 소개
- 인용문 + 개인 경험 연결의 반복 구조
- 다중 인용문 활용 (직접 감정 명명 회피)
- 요약 테이블 + 마무리 인용문
- 토론 질문 (스터디용, 선택)

**구조 예시:**
```markdown
> 스터디 안내 (선택)

## 들어가며
(개인적 배경, 왜 이 책을 읽게 되었는지)
(현재 상황과 연결: 힘든 시기, 회복 중 등)

> 핵심 인용문
>
> — 저자

(인용문이 왜 와닿았는지 - 감정 명명 대신 상황으로)

## 스터디원들과 나누는 질문 (선택)
> 토론 질문들
```

**특징적 패턴:**
- 인용문을 많이 사용하되, 각 인용 후 감정 명명 대신 상황/경험으로 연결
- "~기보다는, 내 모습이 그대로 보였다" 식의 전환 표현
- "선뜻 받아들이기 어려웠다. 그런데 곰곰이 생각해 보면" 식의 저항-수용 패턴
- 수사적 질문: "얼마나 많은 ~를 미루고, 외면하고, 나중으로 떠넘겼는가"
- 열린 결론: "~가 아닐까", "~인 것 같다"

## 4. 특징적 패턴

### 어그로 도입 패턴

독자의 관심을 끄는 자극적 시작 후 괄호로 실제 상황을 해명하는 자조적 유머:

```
> 먼저 어그로 끌어서 미안하다. 나는 15년차 CTO가 아니다.
> (15년째 개발을 하고 있는 건 맞음)
> 그 다음 현재 CTO도 아니다. (작년에 퇴사함.)
```

**활용 시점:**
- 기술 에세이나 경험 공유 글의 도입부
- 제목이 자극적인 경우 해명
- 독자와의 거리감을 좁히고 싶을 때

### 비유/메타포 활용 패턴

복잡한 기술 개념을 친숙한 상황에 비유:

```
"일종의 페어 프로그래밍이랄까. AI가 키보드를 잡고, 나는 옆에서
'어, 거기 그렇게 하면 안 돼' 하면서 방향을 잡아주는 느낌이다."

"마치 레고 블록을 하나씩 쌓듯이, 작은 조각들을 검증하면서 쌓아올린다."

"마치 오케스트라 지휘자처럼, 각 악기(에이전트)가 언제 어떤 역할을
해야 하는지 정의해둔 거다."

"개발자가 코드를 모른다는 건 마치 운전사가 핸들을 안 잡고 있는 것과 비슷하다."
```

**자주 사용하는 비유 영역:**
- 협업/역할 분담: 페어 프로그래밍, 오케스트라 지휘자-연주자
- 점진적 구축: 레고 블록
- 통제/안정감: 운전, 핸들, 내비게이션
- 위임: 리더십, 매니지먼트, PM-개발자 분업
- 조직/팀워크: 스포츠팀 (축구, 감독-선수, 골대, 필드, 드리블, 패스)
- 도구의 가치/효과: 날개(레드불), 슈퍼파워
- 학습/읽기: 영화 보기(흐름 끊김), 스파링 파트너(AI 활용)

### 문장 리듬 패턴

**긴 문장 후 짧은 문장:**
```
"9월은 회사에서 B2G 사업이 일단락되는 시즌이었다. 지나고 보니 여름이 더운지도
모르고 일했던 그 시절이었는데, 그저 당시에는 끝났다는 안도감에 잠깐의 휴식을
가지고 싶었다."

"그게 다였다."
```

**반복 구문 활용:**
```
"무력하고, 무기력하고 할 수 있는 최선을 다하고자 마음을 다잡아도 결국 무너지고 만다."
```

### 인용문 활용 패턴

**영어-한글 병기:**

```markdown
> Not all those who wander are lost
> 방랑자라고 다 길 잃은 것은 아니다

- J.R.R. 톨킨, 반지의 제왕 중
```

**드라마/영화 대사 인용:**
```markdown

> "글쎄요, 사람들은 좋아해요."

*더 베어 시즌3 중*
```

**광고 카피/슬로건 인용 (도구 소개 글):**

- 글 주제와 연결되는 슬로건으로 분위기 설정
- 도구가 주는 가치를 은유적으로 전달

**인용 후 자연스러운 연결 (권장):**

인용문 후 감정을 직접 명명("뜨끔했다", "찔렸다")하기보다, 상황이나 생각으로 간접 연결:

```markdown
# 피해야 할 패턴 (작위적)
> "리더의 역할은 정답을 아는 것이 아니라 정답을 찾는 환경을 만드는 것이다."

뜨끔했다. 나도 그동안 정답을 주려고만 했던 것 같다.

# 권장 패턴 (자연스러운 성찰)
> "리더의 역할은 정답을 아는 것이 아니라 정답을 찾는 환경을 만드는 것이다."

돌아보면 나도 팀원들에게 답을 주려고만 했다. 질문을 던지고 기다리는 게 더 어려웠는데,
그게 오히려 팀의 성장을 막았던 것 같다.

# 권장 패턴 (감정 전환)
> "전력으로 달려도 모자랄 판에 느긋하게 조깅을 즐긴다."

이 문장에서 뜨끔했다기보다는, 내 모습이 그대로 보였다.
긴급한 상황인 줄 알면서도 조깅하듯 느긋하게 걸었다.

# 권장 패턴 (저항 인정 후 수용)
> "이런 일이 내게 일어난 것은 내게 불운이다"라고 말하지 말고...

이 구절을 처음 읽었을 때 선뜻 받아들이기 어려웠다. 힘든 일을 겪으면서
"이건 행운이야"라고 말하기란 쉽지 않다. 그런데 곰곰이 생각해 보면...
```

- 감정 대신 **구체적인 자기 경험**으로 연결
- "돌아보면", "생각해보면", "나도 ~했었는데" 등으로 담담하게 성찰
- 인용이 왜 와닿았는지를 상황으로 보여주기
- **감정을 명명해야 할 경우**: "~기보다는, ~" 패턴으로 전환하여 시각화
- **저항 인정**: "선뜻 받아들이기 어려웠다" 후 "그런데 곰곰이 생각해 보면" 으로 수용

### 강조 방식

- **굵게** - 핵심 키워드, 중요 개념
- 단독 단락 - 감정적으로 중요한 문장
- 인용 블록 (>) - 타인 말, 책/영화 구절, 핵심 원칙 강조
- 이탤릭 (_텍스트_) - 이미지 캡션, 부가 설명

**bold 마커 좌우 띄어쓰기 규칙 (한글 렌더링 이슈 대응):**
- bold(`**`) 마커와 인접 한글 사이에 공백 한 칸 삽입
- 예: `**효율적으로 읽는 방법** 을 아는 사람은`, `**"읽은 후"** 에 쓰는 것이다`
- 이유: 일부 마크다운 렌더러에서 bold 마커가 한글과 붙으면 렌더링이 깨지는 경우가 있음
- 적용 범위: bold 마커 뒤에 한글 조사(을/를/이/가/은/는/에/의/로 등)가 올 때

### 마무리 패턴

**인용문 + 다짐형:**
```markdown
> 인용문
> 한글 번역

- 출처

---

앞으로의 구체적인 계획이나 다음 글 예고
```

**자기 위로형:**
```markdown
주절주절 칭얼칭얼 말이 너무 길었다.
감상에 그만 젖고 이제 다시 Let's hit it.
```

**에필로그/후기형 (인용 블록 활용):**
```markdown
----

> 2025년 회고를 이 글로 대신한다. 나에게 2025년은 리커버리의 시간이었다.
> ...개인적 소회...

> 종교는 없지만, 신이 있다면 항상 기도하고 싶다.
> "저에게서 타인에 대한 노여움을 거둬주시고..."
```

**액션 아이템형:**
```markdown
**Try**

* 침착하게 해야할 것을 하나씩 해나가기
* 조직 동기부여, 업무 파이프라인 개선 포기하지 않기
* 꺾여도 계속가는 마음
```


```
"인생은 항상 쉬운 결정과 어려운 결정을 쥐어주고 하나를 선택하라고 강요한다.

```

```
새로운 출발을 해보자."

"언젠가 또 생각이 많은 날이 온다면, 다시 이 글을 돌이켜 보며 되새기자."

"감상에 그만 젖고 이제 다시 Let's hit it."
```

```
"이 이야기를 보면서 뼈저리게 공감했다. 레거시 시스템이나 정리되지 않은 프로세스를

"채용에서 이런 투명함은 정말 중요하다고 생각한다. 좋은 점만 보여주고 싶은 유혹이
항상 있지만, 결국 나쁜 점을 숨기고 채용한 사람은 금방 떠난다."
```

### 괄호 안 자조적 코멘트

```
"(팬심 전환 속도가 이렇게 빠를 줄이야.)"

"(아니, 엄청 어렵다.)"

"(본능이다.)"

"(지금 생각하면 정말 민망하다.)"

"(결정을 미루는 것도 결정이라는 걸, 그때는 몰랐다.)"
```

### 액션 아이템 형식

```
**액션 아이템:**
1. **"내가 할 수 있나?"가 아니라 "누가 제일 잘할 수 있나?"로 질문 바꾸기.**
2. **결정을 미루지 않기.** 정보가 70% 모이면 결정하고 나머지는 실행하면서 조정하기.
3. **위임 후 간섭하지 않기.** 결과를 리뷰하되, 과정에 일일이 개입하지 않기.
```

### 요약 테이블 마무리

```
| 배운 것 | 액션 아이템 |
|---------|------------|
| 시스템 없이는 성장도 없다 | "우리 팀의 엑셀"을 찾아서 시스템화하기 |
| 고통스러운 변화도 필요하다 | "지금이 유일한 타이밍인가?" 물어보기 |
| 비난 없는 문화 | 리더가 먼저 실패 사례 공유하기 |
```

### 인용문 연결 (책 리뷰)


### 방법론/가이드 표현

```
"기술 아티클을 처음부터 끝까지 한 번에 읽지 마라. 최대 3번에 걸쳐 읽어라."

"5분이면 된다. 길어야 10분. 이 단계의 핵심은 '읽을지 말지'를 빠르게 결정하는 것이다.
근데 정말 5분 만에 판단이 가능한가? 된다."

"내 경우에는 아침에 RSS 피드를 훑으면서 1단계를 수행한다.
10개 중 2~3개만 2단계로 넘어간다. (대부분은 1단계에서 끝난다.)"

"마치 영화를 보다가 모르는 배우가 나올 때마다 검색하면 줄거리를 놓치는 것과 같다."

"3단계에서 AI의 역할은 '스파링 파트너'다.
내가 펀치를 던지고, AI가 받아치고, 그 과정에서 내 이해가 단련된다.
AI가 대신 싸워주는 게 아니다."

"솔직히 말하면 3단계까지 가는 글은 한 달에 몇 편 안 된다. 오히려 그래야 한다."

"그만두는 것도 능력이다. 아무 글이나 끝까지 읽는 것보다 빠르게 걸러내는 게 더 중요하다."

"이 방법을 적용하면서 가장 크게 달라진 건, 읽는 양이 아니라 읽는 태도였다."
```
