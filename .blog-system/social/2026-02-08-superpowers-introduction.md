# 소셜 미디어 포스트

## LinkedIn (전문가 네트워크용)

---

Claude Code에 날개를 달아줬다.

"만들어 줘" 하면 바로 코딩부터 시작하는 AI.
편리하지만, 프로젝트가 커질수록 문제가 된다.

요구사항을 자기 마음대로 해석하고,
테스트 없이 "다 됐어요"라고 하고,
실제로 돌려보면 안 돌아가는 경우.

한두 번이 아니었다.

그래서 interview 커맨드, TDD 스킬, 코드 리뷰 스킬을 직접 만들어 썼는데,
이걸 전부 통합하고 더 강력하게 만든 프레임워크를 발견했다.

Superpowers.
GitHub 4.7만 스타. MIT 오픈소스.

설치하면 이렇게 바뀐다:

질문 -> 설계 -> 계획 -> TDD -> 서브 에이전트 병렬 개발 -> 검증 -> 코드 리뷰

이 7단계가 명령어 없이 자동으로 실행된다.

가장 인상적인 건 설득의 심리학(로버트 치알디니)을 LLM에 적용한 점이다.

1) 권위의 원칙 - 스킬이 필수임을 명시
2) 일관성의 원칙 - 스킬을 따르겠다고 선언하게 만듦
3) 사회적 증거 - 모든 스킬이 이 방식을 따른다고 알려줌

실제 치알디니 본인이 참여한 연구에서 이 원칙이 LLM에도 먹힌다는 게 증명됐다.

실전 사례도 꽤 인상적이다:

- Notion 클론: 45분 개발, 수동 코드 0줄, 테스트 커버리지 87%
- Next.js 16 마이그레이션: 500줄 계획 자동 생성
- 인증 시스템 스킬화: 6개 프로젝트에 적용, 14시간 절감
- 채널톡 연동: 3일 걸릴 작업을 30분에

그리고 Writing Skills라는 메타 스킬로 커스텀 스킬을 직접 만들 수 있다.
회사 코딩 컨벤션 강제, 외부 API 연동, 심지어 프로그래밍 책 PDF를 던져주고
"책 읽고 배운 걸 스킬로 만들어"라고 요청하는 것도 가능하다.

스킬을 쌓아갈수록 점점 강력해진다.
팀의 개발 지식 베이스가 되는 거다.

설치는 두 줄이면 끝난다.

일단 깔고 평소처럼 작업을 시작하면,
Claude가 알아서 질문부터 시작할 거다.

자세한 설치법, 7단계 워크플로우, 실전 팁은 블로그에 정리했다.

## 이미지 추천

1. **7단계 워크플로우 다이어그램** - 블로그에 포함된 mermaid 플로우차트를 깔끔한 인포그래픽으로 변환. 브레인스토밍부터 코드 리뷰까지의 흐름을 한눈에 보여주는 이미지가 가장 임팩트 있음.

2. **Before/After 비교 이미지** - 왼쪽: "만들어 줘" -> 바로 코딩 시작 (X 표시), 오른쪽: "만들어 줘" -> 질문부터 시작 -> 설계 -> TDD... (O 표시). 핵심 변화를 직관적으로 전달.

3. **실전 사례 숫자 카드** - "45분, 0줄, 87%" / "500줄 계획 자동 생성" / "3일 -> 30분" 등 임팩트 있는 숫자를 카드형 이미지로 구성.

4. **설득의 심리학 3원칙 도식** - 권위, 일관성, 사회적 증거가 LLM에 어떻게 적용되는지 간단한 도식으로 표현. 차별화 포인트를 시각화.
