# 소셜 포스트 템플릿

이 템플릿은 `/blog-write approve` 또는 `/blog-write social` 실행 시 자동으로 생성되는 소셜 포스트의 구조를 정의합니다.

## 파일 형식

```markdown
---
blog_post: {original_filename}.md
created_at: {ISO8601 timestamp}
blog_url: "https://blog.flowkater.com/posts/{slug}"
---
```

## 플랫폼별 특성

### LinkedIn
- **길이**: 1000~1500자
- **톤**: 전문적, 인사이트 강조
- **구조**: 훅 → 문제제기 → 해결책 → CTA
- **이모지**: ✅ ❌ 👉 📝 💡 등 적절히 사용
- **해시태그**: 5~10개 (#AI코딩 #개발자 #프로그래밍 등)

### Threads / X
- **길이**: 280~500자
- **톤**: 캐주얼, 대화체
- **구조**: 강한 훅 → 핵심 메시지 → 링크
- **이모지**: 😅 👀 ☕ 등 자유롭게
- **해시태그**: 3~5개

## 훅 유형

1. **고백형**: "솔직히 말하면...", "15년차 개발자의 고백:"
2. **질문형**: "이런 경험 없나요?", "~할 때 뭐 하세요?"
3. **반전형**: "~인 줄 알았는데, 아니었다."
4. **숫자형**: "3가지 이유", "1년간의 실험 결과"
5. **논쟁형**: "~는 틀렸다", "~해야 하는 이유"

## 해시태그 카테고리

### 기술/개발
```
#개발자 #프로그래밍 #개발 #코딩 #테크
#AI코딩 #바이브코딩 #ClaudeCode #TDD
#소프트웨어개발 #개발생산성 #CTO
```

### 회고/에세이
```
#회고 #개발자일상 #개발자이야기
#성장 #자기계발 #스타트업
```

### 독서/리뷰
```
#독서 #책추천 #북리뷰 #개발서적
```

## 사용 방법

1. `/blog-write approve` 실행 시 자동 생성
2. `/blog-write social "{slug}"` 로 수동 생성/확인
3. `/blog-write social-revise "{피드백}"` 로 수정

## 예시

### 입력 (블로그 글 핵심)
- 주제: 바이브 코딩의 불안감과 해결책
- 핵심 메시지: AI가 코드 짜도 인간이 통제력 유지해야
- 해결책: tdd-go-loop 워크플로우

### 출력 (LinkedIn)
```
**바이브 코딩, 정말 편한가요?**

요즘 AI 코딩 도구가 쏟아지고 있습니다.
"한 달 걸릴 일을 3시간 만에!" 같은 이야기가 매일 들립니다.

그런데 저는 묘한 불안감을 느꼈습니다.

- 코드가 돌아가긴 하는데, 내가 이걸 이해하고 있나?
- AI가 열심히 코드 생성하는 동안 뭘 해야 하지?

켄트 백의 '증강형 코딩'에서 힌트를 얻었습니다.
"AI가 코드를 작성하더라도, 인간이 통제력을 유지해야 한다."

그래서 만든 게 `tdd-go-loop`입니다...

👉 상세 내용은 블로그에서: {blog_url}

#AI코딩 #바이브코딩 #TDD #개발자
```

### 출력 (Threads 훅 강조)
```
**15년차 개발자의 고백:**

모든 코드를 AI에게 맡겼더니
생산성은 올랐는데
재미가 사라졌다.

그래서 방법을 바꿨다.

AI는 드라이버, 나는 네비게이터.
체크리스트 하나씩, 직접 리뷰.

느리다. 근데 **내 코드**라는 감각이 생긴다.

📝 {blog_url}
```
