# 소셜 미디어 포스트

## LinkedIn (전문가 네트워크용)

---

**바이브 코딩, 정말 편한가요?**

요즘 AI 코딩 도구가 쏟아지고 있습니다. "한 달 걸릴 일을 3시간 만에!" 같은 이야기가 매일 들립니다.

그런데 저는 개발자로서 묘한 불안감을 느꼈습니다.

- 코드가 돌아가긴 하는데, 내가 이걸 이해하고 있나?
- AI가 열심히 코드 생성하는 동안 뭘 해야 하지?
- Thread 기웃거리다 막상 결과 나오면 집중력은 바닥...

켄트 백의 '증강형 코딩(Augmented Coding)'에서 힌트를 얻었습니다.

**"AI가 코드를 작성하더라도, 인간이 통제력을 유지해야 한다."**

그래서 두 가지를 만들었습니다:

**1️⃣ `/tdd:go` - 생산성과 통제감 둘 다 챙기기**
- 체크리스트 하나씩 실행
- 직접 리뷰하며 코드 파악
- 몰입 상태 유지

**2️⃣ `tdd-go-loop` - 워크플로우 오케스트레이터**
- 자동화할 건 자동화 (codex-review, sql-review 등)
- 여러 하위 에이전트가 병렬로 리뷰
- Tier별 리뷰 강도 자동 조절

핵심은 **내가 직접 봐야 할 코드**와 **자동화해도 되는 검증**을 구분한 것입니다.

솔직히 "힙"하지 않습니다. 바이브하지 않습니다.
하지만 **코드를 이해하면서도 생산성을 챙깁니다.**

CTO 시절에 익힌 감각이 있습니다.
"위임하되 품질을 챙기고, 전체를 파악하되 세부는 신뢰하는 균형."

AI와 일할 때도 똑같이 적용됩니다.

"15년차 CTO가 바이브 코딩하는 방법"
