1. Claude Code의 생산성을 극대화하는 'Superpowers' 프레임워크
Superpowers는 설득의 심리학 기반으로 설계된 Claude Code용 스킬 프레임워크로, 개발 워크플로우를 자동화하여 압도적인 개발 생산성을 제공한다.



1.1. Superpowers의 심리학 기반 설계 원칙
captureSource
제작자 및 개발 배경 

제작자는 퍼(Per) 커뮤니티에서 유명한 개발자 제시핀 센트(Jessy Pincent)이다. 

그는 로버트 치알디니의 설득의 심리학을 LLM에 적용하여 이 스킬 프레임워크를 개발했다. 

치알디니 본인이 공동 저자로 참여한 학술 연구를 통해 설득 원칙이 LLM에도 통한다는 것이 과학적으로 증명되었다. 

설득의 심리학 3가지 원칙 적용 

권위의 원칙: 스킬 파일에 해당 스킬이 필수임을 명시하여, Claude가 반드시 따라야 한다고 인식하게 한다. 

일관성의 원칙: Claude에게 특정 스킬을 따르겠다고 선언하게 만들어, 선언한 내용을 지키도록 유도한다. 

사회적 증거: 다른 스킬들도 모두 이 방식을 따른다고 알려주어, Claude가 이를 표준으로 인식하게 한다. 

압박 시나리오를 통한 스킬 강화 

제작자는 프로덕션 서버 다운 및 분당 5,000달러 손실 상황 등 압박 시나리오를 사용하여 스킬 테스트를 진행했다. 

만약 Claude가 스킬을 사용하지 않고 바로 코딩하면, 테스트를 실패 처리하고 스킬을 더 강화하도록 반복 학습을 시켰다. 

실제로 Claude는 치알디니 연구 논문을 보고 "우리가 이미 설득 원칙을 쓰고 있었다"고 스스로 분석하며, 권위 원칙(Getting Started 스킬)과 일관성 원칙(코드 리뷰 스킬)이 적용되었음을 인지했다. 

이는 슈퍼 AI가 스스로를 분석하고 이해하는 시대가 왔음을 보여준다. 



1.2. Superpowers의 작동 방식 및 설치
captureSource
Superpowers의 핵심 특징 

Superpowers를 사용하면 Claude는 절대로 바로 코딩하지 않고, 무조건 질문부터 시작한다. 

이는 일반 Claude Code가 '만들어 줘' 명령 시 바로 코드를 시작하는 것과 대비된다. 

Superpowers는 질문, 설계, 계획, TDD(테스트 주도 개발), 서브 에이전트 위임, 코드 리뷰, 완성의 흐름을 따른다. 

핵심은 명령어 없이 자동으로 이 개발 흐름이 시작된다는 점이다. 

Superpowers는 Claude Code뿐만 아니라 코덱스나 오픈 코드에서도 사용할 수 있다. 

설치 방법 

Claude Code를 열고 마켓플레이스에 등록한 후 플러그인 인스톨을 진행해야 한다. 

설치 후에도 작동하지 않는다면, Claude를 껐다가 다시 켜야 한다. 

설치가 완료되면 "I have Superpowers"라는 메시지가 보이며, Brainstorm, Write Plan, Execute Plan 스킬을 사용할 수 있게 된다. 



1.3. Superpowers의 자동화된 개발 워크플로우 (7단계)
captureSource
1단계: 브레인스토밍 (Brainstorming) 

프로젝트 요청 시, Claude는 바로 코딩하지 않고 질문부터 시작하여 요구사항을 명확히 한다. 

예: "무엇을 어떻게 만들 건가요?", "대표 문구는 어떻게 쓸까요?", "상황 입력은 자유 텍스트인가요, 카테고리 선택인가요?" 

대화를 통해 기능 목록, 스택, 컴포넌트 구조 등을 포함한 설계 문서를 스스로 작성한다. 

2단계: 워크트리 생성 및 분리 

설계 승인 후, using git worktrees 스킬을 이용해 독립된 워크트리를 만들어 작업을 시작한다. 

이는 실험 중 문제가 생겨도 메인 브랜치가 안전하고, 워크트리만 삭제하면 되므로 Git 꼬임을 방지한다. 

AI 코딩 시 여러 작업을 동시에 진행할 때 워크트리를 분리하는 것이 더 효율적이다. 

3단계: 계획 작성 (Writing Plan) 

Writing Plan 스킬이 설계 문서를 확인하고 구체적인 테스크(Task)로 쪼갠다. 

테스크는 사이즈별로 쪼개지며, Claude는 사용자에게 계획 승인 및 수정 사항 유무를 물어본다. 

4단계: 계획 실행 (Executing Plan) 및 서브 에이전트 활용 

Executing Plan 스킬이 시작되면, 서브 에이전트 드리븐 방식 또는 배치 실행 방식 중 하나를 선택한다. 

서브 에이전트 드리븐 방식이 더 효율적이며, 메인 Claude는 PM 역할을 하며 모니터링하고 서브 에이전트들에게 개발을 시킨 후 결과물을 취합한다. 

이는 단일 Claude로 작업할 때 발생하는 컨텍스트 오류 등의 에러를 줄여준다. 

Dispatch Parallel Agent 스킬을 통해 서브 에이전트들이 UI, 캔버스, API 레이어 등 여러 작업을 동시에 진행하여 시간을 절약할 수 있다. 

5단계: TDD 강제 및 디버깅 자동화 

Superpowers는 TDD(Test-Driven Development) 스킬을 강제한다. 

TDD 사이클: 레드(테스트 작성 및 실패) → 그린(최소 코드 작성 및 테스트 통과) → 리팩토링(코드 정리)을 반복한다. 

예시: 캔버스 합성 로직 구현 시, 텍스트가 똑바로 그려지는지, 폰트 크기가 이미지 너비에 맞게 조절되는지 테스트를 먼저 작성한 후 구현한다. 

TDD를 통해 버그 발생률이 크게 줄어든다. 

시스템 디버깅 스킬: 테스트 실패나 버그 발생 시 4단계 프로세스가 자동으로 발동된다. 

에러 메시지 분석 

관련 코드 범위 특정 

가설 수립 및 검증 

수정 및 테스트 

6단계: 검증 (Verification) 

Verification 스킬을 통해 모든 수정 사항이 고쳐졌는지 확인한다. 

테스트 재실행, 관련 기능 영향 확인, 엣지 케이스(Edge Case) 체크 등을 수행하여, Claude가 "다 됐어요"라고 했지만 실제로는 작동하지 않는 경우를 방지한다. 

7단계: 코드 리뷰 및 피니싱 

테스크가 하나 끝날 때마다 코드 리뷰를 진행한다. 

리뷰 2단계: 스펙 준수 확인 → 코드 품질 확인(보안 취약점, 성능 이슈, 코드 스타일 등) 

리뷰 결과(Critical, Major, Minor 등)에 따라 피드백을 받아 자동으로 코드를 수정한다. 

모든 테스트가 끝나면 Finishing 스킬을 통해 워크트리를 유지할지, GitHub에 PR(Pull Request)을 날릴지 물어보고 워크트리를 정리한다. 



2. Superpowers 실전 활용 데모 및 팁
Superpowers를 활용하여 김동현 명언 생성기 웹 앱을 개발하는 과정을 통해 브레인스토밍부터 TDD 기반 개발, 커스텀 스킬 추가까지의 실질적인 적용 방법을 보여준다.



2.1. 김동현 명언 생성기 웹 앱 개발 시연
captureSource
브레인스토밍 단계 

Claude Code에서 Superpowers 브레인스토밍을 시작하고, "김동현 님 생성기 웹 앱을 만들어 줘"라고 요청한다. 

Claude는 Brainstorming 스킬을 발동하여 경력 멘트 생성 방식(템플릿 기반), 상황 입력 방식(자유 텍스트), 결과물 형태(이미지 민) 등을 질문하며 계획을 작성한다. 

초상권 이슈를 고려하여 사진 대신 AI로 폰트/배경 디자인을 생성하도록 결정한다. 

웹 기술 스택은 Next.js, Vercel, React 기반으로, 이미지 생성은 OpenAI API를 사용하도록 결정한다. 

브레인스토밍 완료 후, 기능, 기술 스택, 컴포넌트 구조가 정리된 설계 문서가 생성된다. 

리서치 및 설계 보강 

초기 명언 템플릿이 재미없다고 판단하여, Claude에게 인터넷에서 김동현 명언을 검색하여 템플릿을 고치도록 요청한다. 

Claude는 나무위키 등에서 자료를 긁어와 최신 트렌드에 맞는 명언을 확보하고, 이를 설계에 반영한다. 

UI/UX 세부 사항(텍스트 스타일, 위치, 배경, 에러 처리, 반응형)까지 설계 문서에 포함시킨 후 승인한다. 

계획 작성 및 실행 

승인 후, Writing Plan 스킬이 발동하여 docs/plans 폴더에 기획 문서를 작성하고 Git 초기화 및 워크트리를 생성한다. 

Writing Plan 스킬은 프로젝트 셋업, 라이브러리, 민 캔버스 등 세부 테스크로 계획을 쪼갠다. 

Claude는 PM 역할을 하고 서브 에이전트들을 세팅하여 Sub-Agent Driven Development 방식으로 실행한다. 

총 8개의 테스크가 설정되며, 각 테스크는 TDD(테스트 작성 후 구현)와 리뷰를 거쳐 개발된다. 

Claude는 5분 40초 동안 혼자서 4개의 테스크를 연속적으로 개발하는 자율 작업을 수행한다. 



2.2. 개발 완료 결과 및 활용 팁
captureSource
개발 결과 

모든 작업이 완료되면, Claude는 커밋 히스토리, 실행 방법, Vercel 배포 방법 등을 안내한다. 

요청했던 폴더 안에 앱 구조, 컴포넌트, 도큐먼트 등 모든 작업이 완료된 코드가 생성된다. 

실제로 앱을 실행하면, 요청한 상황(예: "강의 찍고 있어")에 맞춰 김동현 스타일의 격려 멘트와 미(Meme) 이미지를 생성해 준다. 

Superpowers는 목표 달성을 위해 AI와 반복 작업하는 것보다 계획을 세워 개발하는 좋은 방법이다. 

Superpowers 활용 팁 

토큰 사용량: 계획 단계에서 딥 이터레이션으로 인해 토큰을 많이 사용하므로, Claude Pro나 Max 플랜 사용자에게 추천한다. 

적합한 사용 시점: 간단한 기능 수정보다는 신규 MVP 프로젝트 시작, 복잡한 기능 추가, 전체 코드 수정이 필요할 때 사용하는 것이 좋다. 

자율 작업 가능: 큰 플랜을 잡아 놓으면 몇 시간 동안 혼자서 자율 작업이 가능하다. 

야간 작업 활용: 밤에 계획을 승인시키고 작업을 시키면 아침에 PR이 와 있는 형태로 활용할 수 있다. 



2.3. 커스텀 스킬 추가 및 실제 사례
captureSource
커스텀 스킬 추가 방법 

스킬 라이브러리의 Writing Skills 메타 스킬을 사용하여 기존 스킬을 업데이트하거나 새로운 스킬을 쉽게 생성할 수 있다. 

예: "우리 회사 코딩 컨벤션을 강제하는 스킬을 만들어 줘"라고 요청하면 Claude가 스킬을 만들어 준다. 

새로운 스킬을 만들 때도 TDD, 서브 에이전트, 압박 시나리오 확인 등 기존 개발 프로세스를 똑같이 적용한다. 

프로그래밍 책 PDF 파일을 던져주고 "책 읽고 새로 배운 걸 스킬로 만들어"라고 요청하여 스킬을 생성한 사례도 있다. 

실제 활용 사례: 채널톡 연동 스킬 개발 

채널톡 공식 도큐먼트 전체를 Claude에게 전달하고 Superpowers로 채널톡 연동 스킬을 만들었다. 

이 스킬을 설치하면, 프론트에서 "채널톡 연동 해줘"라고 실행했을 때 Claude가 자동으로 SDK 설치, 봇 설정, 웹훅 연결, ALF(AI-powered Live Feedback) 테스크까지 모두 처리해 준다. 

일반적인 Claude 작업 시 3일 정도 걸릴 것으로 예상되는 작업을 Superpowers를 통해 30분 만에 개발 완료했다. 



2.4. Superpowers 최종 요약
captureSource
심리학 기반 설계: 치알디니의 설득의 6원칙을 기준으로 스킬 준수를 강제한다. 

자동화된 워크플로우: 브레인스토밍부터 TDD, 코드 리뷰까지 설계된 단계를 자동으로 만든다.
